## 🔄 SÜRELİ VEKALET SİSTEMİ (YENİ - KRİTİK)

### Analiz ve Eleştiri

**Mevcut Durum:**
- `ClientPowerOfAttorney` modeli var ama eksik alanlar mevcut
- Süreli vekalet desteği yok
- Avukat-vekalet ilişkisi yok
- Takip açarken vekalet kontrolü yok

**Önerilen İyileştirmeler:**

1. **Vekalet Durumu Enum'u Ekle:**
   - ACTIVE (Geçerli)
   - EXPIRED (Süresi Dolmuş)
   - REVOKED (İptal Edilmiş)
   - PENDING (Onay Bekliyor - portal için)

2. **Vekalet Kapsamı Daha Detaylı Olmalı:**
   - GENEL (Tüm işlemler)
   - ICRA_TAKIP (Sadece icra takipleri)
   - BU_DOSYA (Belirli dosya için)
   - OZEL (Özel kapsam - açıklama ile)

3. **Çoklu Avukat Desteği:**
   - Bir vekalette birden fazla avukat olabilir
   - Her avukatın yetkisi farklı olabilir

4. **OCR'da Süreli Vekalet Tespiti:**
   - "...tarihine kadar geçerlidir" ifadesi
   - "süreli vekalet" ifadesi
   - Tarih formatları (GG.AA.YYYY)

## 🔧 TEKNİK NOTLAR

### Vekalet Geçerlilik Kontrolü Pseudo-Kodu
```typescript
function checkValidPoa(clientId: string, lawyerId: string): boolean {
  const now = new Date();
  
  const validPoa = await prisma.clientPowerOfAttorney.findFirst({
    where: {
      clientId,
      status: 'ACTIVE',
      isActive: true,
      lawyers: {
        some: { lawyerId }
      },
      OR: [
        { isLimited: false }, // Süresiz vekalet
        { 
          isLimited: true,
          validUntil: { gte: now } // Süreli ama henüz dolmamış
        }
      ]
    }
  });
  
  return !!validPoa;
}
```

### Süresi Dolacak Vekaletler Sorgusu
```typescript
async function getExpiringPoas(days: number = 30) {
  const now = new Date();
  const futureDate = new Date();
  futureDate.setDate(futureDate.getDate() + days);
  
  return prisma.clientPowerOfAttorney.findMany({
    where: {
      isLimited: true,
      status: 'ACTIVE',
      validUntil: {
        gte: now,
        lte: futureDate
      }
    },
    include: {
      client: true,
      lawyers: { include: { lawyer: true } }
    }
  });
}
```
